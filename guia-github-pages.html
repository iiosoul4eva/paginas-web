Sube tus HTML a GitHub Pages

Guía paso a paso para publicar tus presentaciones HTML gratis, sin límites y en menos de 10 minutos.

$0 Costo total
∞ Archivos HTML
~8 min Setup inicial
~30 seg Subir cada archivo

1
Crear tu cuenta de GitHub
Solo se hace una vez — 2 minutos
🌐 Instrucciones
1
Ve a github.com y haz clic en "Sign up" (esquina superior derecha).
2
Ingresa tu email, crea una contraseña y elige un nombre de usuario.
⚠️ El nombre de usuario será parte de tu URL. Ej: si eliges "juan123", tu sitio será juan123.github.io
3
Completa la verificación (puzzle) y haz clic en "Create account".
4
Revisa tu email y escribe el código de verificación que te enviaron.
💡
Elige un nombre de usuario corto y profesional. Este será visible en todos tus links.
2
Crear un repositorio
Es como una "carpeta" donde vivirán tus archivos — 1 minuto
📦 Instrucciones
1
Ya logueado, haz clic en el botón "+" (esquina superior derecha) → "New repository".
2
En "Repository name" escribe: mis-presentaciones
(o el nombre que prefieras, sin espacios ni acentos)
3
Selecciona "Public" (es necesario para que GitHub Pages funcione gratis).
4
Marca la casilla "Add a README file" (esto inicializa el repositorio).
5
Haz clic en "Create repository".
⚡
Importante: El repositorio DEBE ser Public. Si lo pones Private, GitHub Pages no funcionará en el plan gratuito.
3
Activar GitHub Pages
Le dices a GitHub que publique tu repositorio como sitio web — 1 minuto
⚙️ Instrucciones
1
Dentro de tu repositorio, haz clic en la pestaña "Settings" (arriba, icono de engranaje).
2
En el menú lateral izquierdo, busca y haz clic en "Pages".
3
En "Source" selecciona "Deploy from a branch".
4
En "Branch" selecciona "main" y la carpeta "/ (root)".
5
Haz clic en "Save".
⏱️
GitHub tarda 1-3 minutos en publicar tu sitio la primera vez. Recarga la página de Settings → Pages y verás un banner verde con tu URL.
4
Subir tu archivo HTML
Arrastra y suelta — 30 segundos
📤 Instrucciones
1
Ve a la página principal de tu repositorio (haz clic en el nombre del repo arriba).
2
Haz clic en "Add file" → "Upload files".
3
Arrastra tu archivo HTML desde tu computadora hacia el área de carga.
También puedes hacer clic en "choose your files" para buscarlo manualmente.
4
En "Commit changes" (abajo), puedes escribir una descripción como "Agrego presentación Cap 1" o dejarlo como está.
5
Haz clic en "Commit changes" (botón verde).
✨
Tip: Renombra tus archivos HTML antes de subirlos con nombres simples sin espacios ni acentos. Ejemplo: cap1-planificacion.html
5
Acceder a tu presentación
Tu URL pública — ¡listo!
🔗 Tu URL será así:
https://tu-usuario.github.io/mis-presentaciones/cap1-planificacion.html
Ejemplo real: Si tu usuario es poker-study y tu repo se llama presentaciones:
https://poker-study.github.io/presentaciones/cap1-planificacion.html
⏱️
Después de subir un archivo, espera 1-2 minutos para que se publique. Si no carga, presiona Ctrl+Shift+R para forzar la recarga.
+
Agregar más presentaciones
Solo repite el Paso 4 cada vez — 30 segundos
🔄 Cada vez que tengas un nuevo HTML:
1
Ve a tu repositorio en github.com
2
"Add file" → "Upload files"
3
Arrastra el nuevo HTML → "Commit changes"
4
Espera 1-2 minutos y accede con la nueva URL
🚀
¡Eso es todo! A partir de ahora, subir una nueva presentación te toma literalmente 30 segundos.
📁
Estructura recomendada
Organiza tus archivos así para mantener todo limpio
📁 mis-presentaciones/
├── README.md
├── index.html ← página principal (opcional)
├── cap1-gestion-tiempo.html
├── cap1-planificacion.html
├── cap2-disciplina.html
├── cap3-mentalidad.html
├── cap4-enfoque.html
└── cap5-trabajo-profesional.html
💡
Tip pro: Si quieres, puedo crearte un index.html que funcione como página principal con links a todas tus presentaciones. Así tendrás un solo link para compartir todo.
💡
Tips y solución de problemas
Todo lo que necesitas saber
📝 Nombres de archivo
  • ✓ Usa minúsculas: cap1-intro.html
  • ✓ Usa guiones en vez de espacios: mi-presentacion.html
  • ✗ NO uses acentos ni ñ: Planificación del Día.html
  • ✗ NO uses espacios: cap 1 intro.html
🔧 Si tu sitio no carga
  • 1 Verifica que el repositorio sea Public
  • 2 Verifica que GitHub Pages esté activado (Settings → Pages)
  • 3 Espera 2-3 minutos después de subir el archivo
  • 4 Verifica que la URL sea correcta (minúsculas, sin espacios)
  • 5 Prueba en ventana de incógnito (Ctrl+Shift+N)
📊 Límites del plan gratuito
  • ✓ 1 GB de almacenamiento por repositorio (más que suficiente)
  • ✓ 100 GB de ancho de banda al mes
  • ✓ Sin límite de archivos HTML
  • ✓ HTTPS gratis incluido
  • ✓ No caduca mientras tu cuenta esté activa
🔄 Actualizar una presentación
1
Sube el nuevo archivo HTML con el mismo nombre que el anterior.
2
GitHub lo reemplazará automáticamente. La URL se mantiene igual.
